  ~~~
  ```
  rig --new-output-component --for-app --name
      --output [--container] [--position]
  ```
  ~~~~~
  ~~~
  ### Description:
  ~~~~~
  ~~~
  Configure a new OutputComponent for the specified App.
  ~~~~~
  ~~~
  ### Flags:
  ~~~~~
  ~~~```--for--app```~~~~~
                ~~~
                The name of the App to configure the new 
                OutputComponent for.
                ~~~~~
                ~~~
                WARNING: The specified App must exist at the path 
                that corresponds to the path assigned to the 
                `--path-to-apps-directory` flag. If the App does 
                not exist, the new OutputComponent will not be
                configured.
                ~~~~~
                ~~~
                Note: It is unnecessary to specify the 
                `--path-to-apps-directory` flag's value, rig 
                will determine an appropriate path at runtime 
                as long as the `--path-to-apps-directory` flag 
                is not specified.
                ~~~
                Note: It is best to let rig determine the value of 
                the `--path-to-apps-directory` flag to insure an 
                appropriate path is used.
                ~~~~~
                ~~~
                Note: More information about the
                `--path-to-apps-directory` flag can
                be obtained via:
                ~~~
                ```
                rig --help path-to-apps-directory
                ```
                ~~~~~
                ~~~~~
                ~~~~~
  ~~~```--name```~~~~~
                ~~~
                An alphanumeric name to assign to the new 
                OutputComponent.
                ~~~~~
                ~~~
                WARNING: The name must be unique. rig will 
                refuse to configure the new OutputComponent 
                if an OutputComponent with the same name 
                already exists.
                ~~~~~
                ~~~
                WARNING: rig will refuse to configure the new 
                OutputComponent if the name is not alphanumeric.
                ~~~~~
                ~~~
                Note: If the name contains spaces, and is not 
                quoted, only the alphanumeric characters up to 
                the first space will be used. For example:
                ~~~
                ```
                rig --new-output-component \
                    --for-app Foo \
                    --name Spaces In Name \
                    --output 'barBaz Foo'
                ```
                ~~~~~
                Would result in the assigned name: `Spaces`
                ~~~~~
                ~~~
                Note: If the name contains spaces, and is 
                surrounded by quotes, then rig will refuse to
                configure the new OutputComponent. For example, 
                the following will fail:
                ~~~
                ```
                rig --new-output-component \
                    --for-app Foo \
                    --name 'Spaces In Quoted Name' \
                    --output 'barBaz Foo'
                ```
                ~~~~~
                ~~~~~
  ~~~```--output```~~~~~
                ~~~
                The output to assign to the new OutputComponent.
                ~~~~~
  ~~~```[--container]```~~~~~
                ~~~
                The name of the container to store the new 
                OutputComponent's data in. This value is used 
                when an App is built for a domain, it determines 
                where in the Domain's storage the OutputComponent's
                data will be located after the App is built.
                ~~~~~
                ~~~
                Defaults to: `OutputComponents`
                ~~~~~
                ~~~
                WARNING: rig will refuse to configure the new 
                OutputComponent if the specified container name 
                is not alphanumeric.
                ~~~~~
                ~~~
                For example, the following will fail:
                ~~~~~
                ~~~
                ```
                rig --new-output-component \
                    --for-app Foo \
                    --name Bar \
                    --container '_Bad-Char$_' \
                    --output 'barBaz Foo'
                ```
                ~~~~~
                ~~~
                Note: If the specified container name contains 
                spaces, and is not quoted, only the alphanumeric 
                characters up to the first space will be used.
                For example:
                ~~~
                ```
                rig --new-output-component \
                    --for-app Foo \
                    --name Bar \
                    --container Spaces In Container \
                    --output 'barBaz Foo'
                ```
                ~~~~~
                Would result in the assigned container: `Spaces`
                ~~~~~
                ~~~
                Note: If the specified container name contains 
                spaces, and is surrounded by quotes, then rig
                will refuse to configure the new OutputComponent. 
                For example, the following will fail:
                ~~~
                ```

                rig --new-output-component \
                    --for-app Foo \
                    --name Baz \
                    --container 'Spaces In Container' \
                    --output 'barBaz Foo'
                ```
                ~~~~~
                ~~~~~
  ~~~```[--position]```~~~~~
                ~~~
                The position to assign to the OutputComponent.
                This position will be used to sort the
                OutputComponent relative to other OutputComponents 
                and DynamicOutputComponents that are served by the 
                same Response or GlobalResponse.
                ~~~~~
                ~~~
                Defaults to: `0`
                ~~~~~
  ~~~
  ### Examples:
  ~~~~~
  ~~~
  ```
  rig --new-output-component \
      --for-app Foo \
      --name BazFooBar \
      --output Bar

  rig --new-output-component \
      --for-app Foo \
      --name BazFooBaz \
      --output Baz \
      --container FooOutput

  rig --new-output-component \
      --for-app Foo \
      --name BazFooBin \
      --output Bin \
      --position 2.4
  ```
  ~~~~~
