  ~~~
  ```
  [--path-to-apps-directory]
  ```
  ~~~~~
  ~~~
  This flag can be used in conjunction with other rig flags to 
  manually set the roady Apps directory path used by rig to an 
  alternative Apps directory path.
  ~~~~~
  ~~~
  Typically it is not necessary to specify this flag. rig will 
  attempt to locate the Apps directory at the following path 
  relative to rig's root directory: 
  ~~~~~
  ~~~
  ```
  ../../../Apps
  ```
  ~~~~~
  ~~~
  If rig cannot locate the expected Apps directory rig will use the
  path to rig's `tmp` directory as the `--path-to-apps-directory`.
  ~~~~~
  ~~~
  rig's tmp directory will be located in rig's root directory, 
  for example: 
  ~~~~~
  ~~~
  ```
  /path/to/rig/tmp
  ```
  ~~~~~
  ~~~
  WARNING: It is best to let rig determine the 
  `--path-to-apps-directory` flag's value to insure 
  an appropriate path is used. 
  ~~~~~
  ~~~
  WARNING: Using the `--path-to-apps-directory` flag in conjunction 
  wtih the `--configure-app-output` flag does work properly at the
  moment. This bug will be addressed. See issue #100:
  ~~~~~
  ~~~
  ```
  https://github.com/sevidmusic/rig/issues/100
  ```
  ~~~~~
  ~~~
  ### Examples:
  ~~~~~
  ~~~
  ```
  rig --new-app \
      --name Foo \
      --path-to-apps-directory ~/AlternativeAppsDirectory

  rig --new-output-component \
      --for-app Foo \
      --name Bar \
      --output 'FooBarBaz' \
      --path-to-apps-directory ~/AlternativeAppsDirectory

  rig --new-request \
      --for-app Foo \
      --name Baz \
      --path-to-apps-directory ~/AlternativeAppsDirectory

  rig --new-response \
      --for-app Foo \
      --name Bazzer \
      --path-to-apps-directory ~/AlternativeAppsDirectory
  
  rig --assign-to-response \
      --for-app Foo \
      --output-components Bar \
      --requests Baz \
      --response Bazzer \
      --path-to-apps-directory ~/AlternativeAppsDirectory
  ```
  ~~~~~
