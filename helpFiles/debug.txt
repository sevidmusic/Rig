  ~~~
  ```
  rig [FLAG] --debug [flags] [options]
  ```
  ~~~~~
  ~~~
  Show debug information for the current call to rig.
  ~~~~~
  ~~~
  The following arguments can be specified to control what 
  debug information is shown, at least one is expected.
  ~~~~~
  ~~~
  ### `flags`
  ~~~~~
            ~~~
            Specifying `flags` will cause rig to output the 
            flags passed to rig and their arguments.
            ~~~~~
  ~~~
  ### `options`
  ~~~~~
            ~~~
            Specifying `options` will cause rig to output any 
            options passed to rig, i.e, the arguments that 
            were specified that are not associated with a 
            flag.
            ~~~~~
            ~~~
            Note: The 'options' argument is provided for 
            future use, at the moment rig does not accept 
            any `options`, only `flags` and flag arguments.
            ~~~~~
  ~~~
  ### Examples:
  ~~~~~
  ~~~
  ```
  rig --help --debug flags options

  rig --new-app --name Foo --debug flags

  rig --start-server --debug options
  ```
  ~~~~~
