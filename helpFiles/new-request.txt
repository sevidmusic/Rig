rig --new-request --for-app --name [--relative-url]

Description:

Configure a new Request for the specified App.

Flags:

--for-app      The name of the App to configure the Request for.

               WARNING: The specified App must exist, if it does not, the new
               Request will not be configured.

--name         A name to assign to the new Request.

               WARNING: The name must be unique, if a Request already
               exists with the same name the new Request will not be
               configured.

               WARNING: The name must be alphanumeric with no spaces, if spaces
               or non-alphanumeric characters are present in the specified name,
               only the alphanumeric characters up to the first space or
               non-alphanumeric character will be used.

               For example:

                   rig --new-request \
                   --for-app Foo \
                   --name 'Spaces & Bad Chars'

               Would result in the assigned name:

                   Spaces

               It is also possible that rig will refuse to configure the Request
               if an alphanumeric name cannot be derived from a name that
               contains spaces or non-alphanumeric characters. This may happen,
               for instance, if the specified name starts with a non-alphanumeric
               character.

--relative-url A relative url to assign to the new Request. This relative url
               will correspond to a complete url when an App is built for a domain.
               By default, the relative url '/' is assigned which will correspond
               to the domain's root url.

               For example:

                   '?request=Foo' corresponds to http://{domain}/?request=Foo

               Defaults to relative root url:

                   '/' the default corresponds to http://{domain}/

Examples:

rig --new-request --for-app Foo --name Bar

rig --new-request --for-app Foo --name Baz --relative-url "?request=Baz"

rig --new-request --for-app Foo --name Bin --relative-url "?Foo=bar&baz=biz"
