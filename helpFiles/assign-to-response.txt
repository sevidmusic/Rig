rig --assign-to-response --for-app --response [--dynamic-output-components]
     [--output-components][--requests]

Description:

Assign one or more Requests, OutputComponents, or DynamicOutputComponents to an
existing Response or GlobalResponse for an existing Darling Data Management
System App.

Flags:

--for-app                    Name of the App that defined the target Response or
                             GlobalResponse.
                             Note: The App must already exist.

--response                   Name of the target Response or GlobalResponse.
                             Note: The Respnse or GlobalResponse must already
                             be defined by the specified App.

--dynamic-output-components  Names of the DynamicOutputComponents to assign to
                             the Response or GlobalResponse.
                             Note: The DynamicOutputComponents must already be
                             defined by the specified App.

--output-components          Names of the OutputComponents to assign to the
                             Response or GlobalResponse.
                             Note: The OutputComponents must already be defined
                             by the specified App.

-requests                    Names of the Requests to assign to the Response or
                             GlobalResponse.
                             Note: The Requests must already be defined by the
                             specified App.

Examples:

rig --assign-to-response \
     --for-app Foo \
     --response Bar \
     --dynamic-output-components ZorFoo BinFoo \
     --output-components Yaz Naz \
     --requests BarBaz Baz
