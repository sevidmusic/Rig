  ~~~
  # Requests
  ~~~~~
  ~~~
  A Request represents a request to a domain.
  ~~~~~
  ~~~
  New requests can be configured for an App via: 
  ~~~~~
  ~~~
  ```
  rig --new-request
  ```
  ~~~~~
  ~~~
  Note: More information about `rig --new-request` can
  be obtained via: 
  ~~~
  ```
  rig --help --new-request
  ```
  ~~~~~
  ~~~~~
  ~~~
  A Request can be assigned to a Response or GlobalResponse. 
  When a Request is assigned to a Response or GlobalResponse,
  the respective Response or GlobalResponse will then configured 
  to respond to the assigned Request:
  ~~~~~
  ~~~
  To assign a Request to a Response or GlobalResponse use:
  ~~~~~
  ~~~
  ```
  rig --assign-to-response
  ```
  ~~~~~
  ~~~
  Note: More information about `rig --assign-to-response` can
  be obtained via:
  ~~~
  ```
  rig --help --assign-to-response
  ```
  ~~~~~
  ~~~~~
  ~~~
  ### Example: Create an App and assign a Request to a Response
  ~~~~~
  ~~~
  ```
  rig --new-app \
      --name Foo

  rig --new-response \
      --for-app Foo \
      --name Baz

  rig --new-request \
      --for-app Foo \
      --name Bar \
      --relative-url 'index.php'

  rig --assign-to-response \
      --for-app Foo \
      --response Baz \
      --requests Bar 
  ```
  ~~~~~
  ~~~
  Note: It is not necessary to assign Requests to GlobalResponses
  since GlobalResponses respond to all requests.
  ~~~~~
  ~~~
  Note: More information about Responses can be obtained via:
  ~~~
  ```
  rig --help Responses
  ```
  ~~~~~
  ~~~~~
  ~~~
  Note: More information about GlobalResponses can be obtained via:
  ~~~
  ```
  rig --help GlobalResponses
  ```
  ~~~~~
