  ~~~
  # Requests
  ~~~~~
  ~~~
  A Request represents a request to a domain, and conforms to the 
  following roady interface:
  ~~~~~
  ~~~
  ```
  <?php
  
  namespace roady\interfaces\component\Web\Routing;
  
  use roady\interfaces\component\SwitchableComponent as SwitchableComponentInterface;
  
  /**
   * A Request is a SwitchableComponent that represents a specific 
   * request to a domain.
   *
   * Methods:
   *
   * public function getState(): bool;
   * public function switchState(): bool;
   * public function getType(): string;
   * public function export(): array<string, mixed>;
   * public function import(array<string, mixed> $export): bool;
   * public function getName(): string;
   * public function getUniqueId(): string;
   * public function getLocation(): string;
   * public function getContainer(): string;
   * public function getGet(): array;
   * public function getPost(): array;
   * public function getUrl(): string;
   *
   */
  interface Request extends SwitchableComponentInterface
  {
  
      /**
       * Return the $_GET array for this Request. 
       *
       * @return array<mixed> Return the $_GET array for this Request. 
       */
      public function getGet(): array;
  
      /**
       * Return the $_POST array for this Request. 
       *
       * @return array<mixed> The $_POST array for this Request. 
       */
      public function getPost(): array;
  
      /**
       * Return the url associated with this Request.
       *
       * @return string The url associated with this Request.
       */
      public function getUrl(): string;
  
  }
  ```
  ~~~~~
  ~~~
  New requests can be configured for an App via: 
  ~~~~~
  ~~~
  ```
  rig --new-request
  ```
  ~~~~~
  ~~~
  Note: More information about `rig --new-request` can
  be obtained via: 
  ~~~
  ```
  rig --help --new-request
  ```
  ~~~~~
  ~~~~~
  ~~~
  A Request can be assigned to a Response or GlobalResponse. 
  When a Request is assigned to a Response or GlobalResponse,
  the respective Response or GlobalResponse will then configured 
  to respond to the assigned Request:
  ~~~~~
  ~~~
  To assign a Request to a Response or GlobalResponse use:
  ~~~~~
  ~~~
  ```
  rig --assign-to-response
  ```
  ~~~~~
  ~~~
  Note: More information about `rig --assign-to-response` can
  be obtained via:
  ~~~
  ```
  rig --help --assign-to-response
  ```
  ~~~~~
  ~~~~~
  ~~~
  ### Example: Create an App and assign a Request to a Response
  ~~~~~
  ~~~
  ```
  rig --new-app \
      --name Foo

  rig --new-response \
      --for-app Foo \
      --name Baz

  rig --new-request \
      --for-app Foo \
      --name Bar \
      --relative-url 'index.php'

  rig --assign-to-response \
      --for-app Foo \
      --response Baz \
      --requests Bar 
  ```
  ~~~~~
  ~~~
  Note: It is not necessary to assign Requests to GlobalResponses
  since GlobalResponses respond to all Requests.
  ~~~~~
  ~~~
  Note: More information about Responses can be obtained via:
  ~~~
  ```
  rig --help Responses
  ```
  ~~~~~
  ~~~~~
  ~~~
  Note: More information about GlobalResponses can be obtained via:
  ~~~
  ```
  rig --help GlobalResponses
  ```
  ~~~~~
